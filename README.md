# Mini NPU Simulator

## 1. 프로젝트 개요

본 프로젝트는 NPU(Neural Processing Unit)의 기본 연산인 MAC(Multiply-Accumulate)을 기반으로, 입력 패턴과 필터(Cross, X)를 비교하여 패턴을 분류하는 시뮬레이터를 구현하는 것을 목표로 한다.

사용자는 직접 데이터를 입력하거나 JSON 데이터를 활용하여 연산을 수행할 수 있으며, 성능 분석 및 최적화 기능까지 포함한다.


---

## 2. 실행 방법

```bash
python main.py
```

---


## 3. 주요 기능

### 3.1 MAC 연산

* 동일 위치의 값끼리 곱한 후 모두 더하는 연산 수행
* Cross 필터와 X 필터 각각에 대해 점수 계산

---

### 3.2 패턴 분류

* Cross 점수 > X 점수 → **Cross**
* X 점수 > Cross 점수 → **X**
* 두 값이 거의 동일 → **UNDECIDED**

---

### 3.3 부동소수점 비교 정책

부동소수점 연산의 오차를 고려하여 epsilon 기반 비교를 사용한다.

```python
abs(score_a - score_b) < 1e-9
```

---

### 3.4 모드 구성

#### 모드 1: 사용자 입력

* 3×3 필터 및 패턴 입력
* 입력 검증 수행

  * 행/열 개수 확인
  * 숫자 파싱 검증
  * 오류 시 재입력 유도

#### 모드 2: JSON 데이터 분석

* data.json에 정의된 패턴 및 필터 자동 로드
* 다양한 크기(3, 5, 13, 25 등) 지원
* PASS / FAIL 결과 출력

#### 모드 3: 최적화 성능 비교

* 2차원 배열 vs 1차원 배열 성능 비교
* 동일 입력 / 동일 반복 조건에서 측정

#### 모드 4: 패턴 생성기

* 입력 크기 N에 대해 Cross / X 패턴 자동 생성
* 생성된 패턴을 다른 모드에서 재사용 가능

---

## 4. 성능 분석

### 4.1 시간 복잡도

MAC 연산은 다음과 같은 구조를 가진다:

```python
for i in range(N):
    for j in range(N):
        total += pattern[i][j] * filter[i][j]
```

* 총 연산 횟수: N × N = N²
* 시간 복잡도: **O(N²)**

---

### 4.2 측정 결과 특징

* 입력 크기가 증가할수록 실행 시간은 제곱 형태로 증가
* 3×3 → 5×5 → 13×13 → 25×25 순으로 성능 감소 확인

---

## 5. 최적화 (보너스)

### 5.1 1차원 배열 변환

* 2차원 배열을 1차원으로 변환하여 메모리 접근 단순화
* 캐시 효율 향상 기대

### 5.2 성능 비교

* 동일 입력, 동일 반복 횟수로 측정
* 1차원 접근이 약간 더 빠른 경향 확인

---

## 6. 패턴 생성기 (보너스)

* 크기 N 입력 시 Cross / X 패턴 자동 생성
* 생성된 패턴은 전역 변수로 저장되어 재사용 가능
* 모드 1 및 성능 분석에서 활용

---

## 7. 실패 원인 분석

본 프로젝트의 FAIL 케이스는 다음 세 가지 유형으로 구분된다.

---

### 7.1 데이터 라벨링 오류

- 실제 패턴과 expected 값이 불일치
- 예: `size_5_4`

---

### 7.2 분류 경계 문제 (UNDECIDED 발생)

- Cross와 X 점수가 유사하여 명확한 분류 불가
- epsilon 기반 비교로 인해 UNDECIDED 반환
- 예: `size_3_float_1`, `size_5_undecided`, `size_5_undecided2`

---

### 7.3 의도된 예외 테스트 케이스

- 필터 누락 또는 크기 불일치
- 예: `size_10_1`, `size_5_mismatch_1`
